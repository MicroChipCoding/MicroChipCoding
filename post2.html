Blog Post #2: HTML Basics (Part 2/2)

By MicroChipCoding, posted on 11/22/24 at 4:50 PM (PST)
Topic: Coding

Welcome to my 2nd blog post! This is a continuation from the first post about HTML. You will learn more advanced mechanics in HTML (the mechanics I know) in this post.

HTML has text and heading levels, images, structural components, but you're gonna need a lot more than that to build a cohesive website. Let's get to some additional information here. However, if you do not know the important components of HTML or if you're lost, check out Part 1 of HTML Basics first.

The List: Part 2

  • The a tag

    • The a tag is an anchor element that allows other elements to be clickable links. You need to wrap the designated element inside the a element and add an href element with the value of the designated link. If you want the link to pop up in a new tab, add the attribute target with a value of _blank.

  • The br tag (self-closing/empty)

    • The br tag is a line break, allowing you to force text to move to the line below. Add a br element right before text that you want to be on a new line.

  • The hr tag (self-closing/empty)

    • This tag, or the horizontal rule, acts as a horizontal line divider that can be placed below elements.

  • The form tag

    • You can create forms with this tag Simply add the action attribute with its value being the link where the form data is sent. We'll get to that later.

  • The input tag (self-closing/empty)

    • This is where users can input data into the form. A few input types (not all) include radio (multiple-choice questions), checkbox (multiple answers), and text. You need a name and value attribute for most input types (not text). The name should be the same for every answer option in a question and the value attributes should be different per answer.

  • The button tag

    • At the end of a form, for users to be able to submit the form, you need a button with a type attribute of the value submit. Inside the button, you can choose what text to display.

  • The ol, ul, and li tags

    • Use the ol tag to create ordered lists (with numbers) and the ul tag to create unordered lists (in bullet points). Inside the lists, add li (list point) tags which are the numbers or bullets.

  • The b, i, and u tags

    • Wrap text around the b tag to make it bold, the i tag to make it italic, and the u tag to make it underlined.

  • Comments

    • To add comments in HTML, you need the opening tag < ! - - and the closing tag - - > (no spaces though). Comments hide text from the webpage and only appear to web developers.

  • Format for href attributes in anchor tags (especially in the nav tag)

    • For the value of href, you must put its file name with a forward slash /. Inside the a tag, you can add text as usual.

And this wraps up my 2nd blog post! I hope you enjoyed learning HTML as much as I did. The next post will be about a coding language that is linked to HTML but used to make sites like appealing, that being the one and only CSS. I will post it on Tuesday!
