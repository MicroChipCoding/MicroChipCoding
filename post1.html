Blog Post #1 - I LOVE CODING! (HTML)

By MicroChipCoding, posted on 11/15/24 at 8:20 PM (PST)
Topic: Coding

Welcome to my first ever blog post! Sorry for posting this late, I was playing badminton with my friends and I had a meeting to go to! In this post, I will share some important tips to get started with the coding language HTML.

HTML stands for Hypertext Markup Language. HTML is a markup language because you can edit the code to allow your website and webpages have a wide variety of structural components. So let's get started, shall we?

Here is a list of many HTML tags that you will need if you're interested in coding your own websites (which I highly recommend instead of using an online website builder w/o code):

  • Before we start, make sure that every element mentioned here has opening tags <> and closing tags except for the elements that are self-closing/empty. You will also need to do something called nesting, which is placing an element inside of another element.
  • Some HTML tags have values in the opening tag called attributes. They are used to further customize the element. Attributes are formatted like this: attribute="value".
  • All content inside opening and closing tags must be lowercase.
  • The declaration

    • You need to add this tag at the start of every HTML webpage. This tells the browser that the page is being programmed with HTML.

  • The html, head, body, main, section, and footer tags (Page Structure Tags)

    • The HTML tag is required as the root of the webpage. You must nest the head, body, and footer tags in the html tag. Use the attribute lang and its value en (lang="en") to display the website in English. It is optional but recommended to nest in a main element in the body as well as different section elements in the body for improving Search Engine Optimization (SEO for short).

  • Meta elements and title

    • In the head element, you have to add a title for the webpage with the title element. The text inside tells the browser to display the text on the tab. You also need 2 meta elements. The first should have an attribute called charset and a value of UTF-8. The purpose of this is to tell the browser to encode characters in this widely used format, that being the Unicode Transformation Format (8 bits). The second is not needed but HIGHLY recommended. It should have 2 attributes: name (value = viewport) and content (value = width=device-width, initial-scale=1.0). This attribute is great for optimizing how the website displays on mobile screens.

  • The nav tag

    • The nav tag allows you to add multiple webpages in your websites. This tag goes in the head element of the website. Note that the elements I mentioned above including the nav element must be in all web files, and html files end in .htm or .html. Use this element to insert hypertext references to other HTML files or webpages.

  • The h1 tag

    • This tag is used for headings. There are 6 levels of headings going from h1 to h6, with h1 being the highest level that has the biggest font size. I recommend using just 1 HTML tag as the title of each webpage.

  • The p tag

    • This tag is used for paragraphs. You can use the p tag to allow your webpages to display general text. Add as many p tags as you need.

  • The img tag (self-closing/empty)

    • This tag is used for images. After you add the img tag, in, you need to add the attribute src. This attribute should be displayed like this: src="(image link"). You can add an optional but recommended attribute called alt="(text that would appear if the image fails to load)".

And there you have it! I hope you enjoyed! These are just a few basic HTML tags and there is a lot more to learn! But why should you learn HTML? HTML is a very popular coding language that is easy to learn and has many elements. You can create websites for free on platforms like Wix, WordPress, and even GitHub (which I recommend for code editing). I think learning how to code is a useful skill that can potentially be lucrative with lots of practice and expertise. With all that being said, if you want to learn more about coding, check out the sites on my About Me page.
