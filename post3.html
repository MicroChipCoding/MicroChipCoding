Blog Post #2: CSS Formatting (Part 1/2 of CSS Basics)

By MicroChipCoding, posted on 11/26/24 at 2:00 PM (PST)
Topic: Coding

Without CSS, or Cascading Style Sheets, today's websites would not look appealing and they would also not be user-friendly. In this post, let's learn about the general structure of CSS.

Before we get into the good stuff, make sure to create a file in your code editor that ends with .css (styles.css is the default name). Next, add a link element in the header as well as the attribute rel with a value of stylesheet and the attribute href with a value of your CSS file name. If you don't know how to do this, go to Post 1 to learn how to add HTML attributes.

Each CSS "selector" allows you to choose an element, class, ID, etc. Classes are attributes that can be assigned to a select group of elements (attribute: class) and unique IDs can only be assigned to 1 element per ID.
CSS selectors have 3 lines:

  1. The element, class, or ID

    2. Add at least 1 element by simply typing the element name. I only recommend this for elements that you plan to keep the same for every webpage as you can customize HTML elements in many different ways. For classes, same thing but you have to add a period (.) before each class. For IDs, add a hashtag (#) before each ID. If you want to add more than 1 selector on the same line, add commas after each selector but you can only have 1 selector type per line (ex: no elements and classes together). Make sure to add a left bracket ( after all selectors are inputted.

  2. The CSS property

    3. CSS has many properties for you to customize elements, such as text-align, margin, padding, font-family, color, width, and height. Alongside most properties, you will also need a value assigned to it. This is called a property-value declaration which looks like this: property: value; . Add as many properties as you wish per group of selectors or individual selector.

  3. Just add a right bracket } to close the selector off
